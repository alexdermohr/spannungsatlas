# Spannungsatlas – UX/UI Blaupause

Dieses Dokument enthält die konzeptionellen UI- und UX-Ideen für den Spannungsatlas, die den epistemischen Reflexionsraum visuell und interaktiv abbilden.

## 1. Leitprinzip

UI ist kein Formular, sondern ein:
→ **Reflexionswerkzeug mit konkreten Interaktionsregeln**
Das System zwingt Denken, verhindert stille Glättung und macht Irritation produktiv.

## 2. Hauptansicht: „Reflexionsraum“

Zonen müssen in folgender zwingender Reihenfolge durchlaufen werden:
1. Situation
2. Beobachtung
   → UI-Check: „Ist diese Beschreibung rein beobachtbar (ohne Interpretation)?“ (Kamera-Test)
   → Bei Verneinung: Hinweis „Formulierung enthält Deutung – bitte trennen“
3. Deutung (erst möglich, wenn Beobachtung vorhanden)
   → Jede Deutung zeigt sichtbar ihren Evidenztyp (beobachtungsnah, abgeleitet, spekulativ)
4. Gegen-Deutung (zwingend nach Deutung)
5. Unsicherheit (zwingend vor Abschluss)
6. Spannungsnetz

👉 Jede Zone ist visuell getrennt


## 3. Spannungsnetz (zentrales Element)

Graph:
- **Knoten** = Personen / Faktoren
- **Kanten** = Spannungen

→ Kanten müssen zwingend enthalten:
- Kontext (Situation)
- Zeitbezug
- Richtung (wer wirkt auf wen)

👉 Verhindert abstrakte „Spannungswolken“.

→ Wenn ähnliche Spannungen mehrfach auftreten:
UI zeigt Hinweis: „Mögliche Musterbildung – prüfen, ob kontextabhängig“

👉 Nutzer sieht:
- nicht „Person ist schwierig“
- sondern: „Spannung X zwischen A und B“

## 4. Drift-Ansicht (entscheidend)

Zeigt als Vergleichsansicht (nicht nur Timeline):
- WAS sich geändert hat
- WARUM (neue Beobachtung / Intervention / Kontext) inkl. Klassifikation

👉 Beispiel:
Zeit →
T1: „Ali provoziert absichtlich“
T2: „Ali reagiert auf Unsicherheit“
T3: „Beides möglich“

## 5. Widerspruchs-UI

Das System erzwingt mindestens eine Gegen-Deutung bei jeder Deutung.

→ System prüft:
   • bezieht sich Gegen-Deutung auf gleiche Beobachtung?
   • enthält sie eine echte alternative Erklärung?

👉 Wenn nicht: Hinweis anzeigen „Keine echte Gegen-Deutung – bitte alternative Erklärung formulieren“.
👉 Wenn keine gültige Gegen-Deutung existiert: Blockiere Abschluss.

## 6. Unsicherheits-UI

Nicht nur ein Slider, sondern:
- Jede Deutung benötigt zwingend einen Unsicherheitsgrad
- Jede Deutung benötigt eine kurze, verpflichtende Begründung

👉 Ohne Begründung: keine Speicherung.

## 7. Multi-Perspektiven-Modul (Erweiterung)

- Mehrere Fachkräfte reflektieren dieselbe Situation
- Perspektiven dürfen NICHT aggregiert werden
- Unterschiede und Abweichungen in der Deutung bleiben visuell nebeneinandergestellt und sichtbar
